# Operator inspects working-tree diff

This journey covers the as-built operator path through the [groom dashboard](../gui/screens/groom-dashboard.md) for inspecting one workflow checkout's working-tree changes. It can start from the selected worker's [working-tree diff disclosure](../gui/screens/groom-dashboard.md#detail-working-tree-diff-toggle), which fetches the default checkout diff directly into the detail pane, or from the dashboard Diff activity mode, which uses the shared repository picker to choose a container/repository, loads [workspace diff data](../workspace-diff-data.md) through [GET /diff/{container_id}](../http/groom.md#get-workspace-diff), renders a changed-file tree through the [dashboard diff file tree](../dashboard-diff-file-tree.md), and renders one chosen parsed file through Diff2Html. Server-side diff reads prefer [sidecar live sessions](../sidecar-live-sessions.md) `getDiff` RPCs and fall back to the workflow workspace-volume diff reader when no live sidecar can serve the data.

- start: the groom server is running, the browser has loaded the [groom dashboard](../gui/screens/groom-dashboard.md), and at least one [workflow container](../concepts/workflow-container.md) may have a known `workspace_volume`. For the detail-disclosure path, a worker with at least one open gate is selected in the inbox detail pane. For the Diff pane path, the dashboard may start with no selected repository, or with [dashboard selected repository state](../dashboard-selected-repository-state.md) already populated from an earlier Files or Diff pane selection.
- steps:
  1. Optional detail entry: the operator expands [toggle detail working tree diff](../gui/screens/groom-dashboard.md#toggle-detail-working-tree-diff) from the selected worker detail pane. The browser wires the disclosure after the detail fragment swap, ignores closing events and previously loaded disclosure bodies, shows `Loading diff...` for the first expansion, and sends `GET /diff/{container_id}` without a `repo` query so the endpoint uses its default checkout selection.
  2. [Serve workspace diff](../http/groom.md#serve-workspace-diff) handles the detail-disclosure request or any later Diff pane request. It first asks the connected sidecar, when present, for `getDiff` with the selected `repo` value, returns the successful RPC result's `diff` member as `text/plain`, and otherwise falls back to the workflow's known workspace volume through the [workspace volume diff reader](../concepts/workspace-volume-diff-reader.md). Unknown workflow state, missing volume metadata, sidecar RPC failure, falsey sidecar diff, no checkout, git failure, timeout, or empty working tree all become a successful empty text response rather than an endpoint-specific error body.
  3. For the detail-disclosure path, the browser reads the fulfilled diff response as text without checking `response.ok`, marks the disclosure body loaded, and either renders non-whitespace unified diff text through Diff2Html with `drawFileList: true`, line matching, line-by-line output, and dark color scheme, or renders `(no changes)`. A fetch or body-read rejection writes `failed to load diff` and leaves the disclosure retryable. This path ends inside the worker detail pane and does not select a repository or changed-file row.
  4. Primary Diff pane entry: the operator activates [select activity diff mode](../gui/screens/groom-dashboard.md#select-activity-diff-mode) from the activity bar. The dashboard sets `.app[data-mode]` to `diff`, marks the Diff activity control active, closes the shared repository menu if it was open, and enters the Diff pane loader.
  5. If no selected repository container exists, [select activity diff mode](../gui/screens/groom-dashboard.md#select-activity-diff-mode) renders `Pick a container / repo above.` in `#diff-tree` and returns without changing `#diff-view`, issuing a diff request, parsing diff text, or rendering changed-file rows. If a repository is already selected, the same interaction resets `#diff-tree` to `Loading changes...`, resets `#diff-view` to `Select a changed file to see its diff.`, and sends `GET /diff/{container_id}?repo={repo}` using the currently selected container id and repository path.
  6. When a repository must be chosen, the operator activates [open diff repository picker](../gui/screens/groom-dashboard.md#open-diff-repository-picker). The browser positions and opens the shared repository menu, replaces `#repo-menu` with `Loading...`, clears and focuses [repository menu search input](../gui/screens/groom-dashboard.md#repository-menu-search-input), and fetches [GET /repos](../http/groom.md#get-repository-menu).
  7. [Serve repository menu](../http/groom.md#serve-repository-menu) reads the current in-memory workflow registry, keeps workflows with known workspace volumes, resolves checkout directories through the [workspace volume repository-directory reader](../concepts/workspace-volume-repository-directory-reader.md), and returns [repository menu data](../repository-menu-data.md) rendered as [repository menu option](../gui/screens/groom-dashboard.md#repository-menu-option) rows. Individual checkout-discovery failures become root options or absent entries for that workflow rather than a failed menu endpoint response.
  8. The dashboard inserts the repository option HTML into `#repo-menu`. The operator may use [filter repository menu options](../gui/screens/groom-dashboard.md#filter-repository-menu-options), which hides or shows already-loaded `.repo-item` rows by case-insensitive `data-label` matching without requesting new data, selecting an option, or changing the active pane.
  9. The operator activates [select repository menu option](../gui/screens/groom-dashboard.md#select-repository-menu-option). The dashboard stores the option's `data-container`, normalized `data-repo`, and `data-label` in [dashboard selected repository state](../dashboard-selected-repository-state.md), updates every repository picker label, dispatches the [dashboard active pane loader](../concepts/dashboard-active-pane-loader.md), and closes the repository menu.
  10. Because the active mode is `diff`, [select repository menu option](../gui/screens/groom-dashboard.md#select-repository-menu-option) follows the Diff pane load path: it sets `#diff-tree` to `Loading changes...`, resets `#diff-view` to `Select a changed file to see its diff.`, and sends `GET /diff/{container_id}?repo={repo}` to [get workspace diff](../http/groom.md#get-workspace-diff) with the selected container id and selected repository path URL-encoded from browser state.
  11. The browser treats any fulfilled diff response as [workspace diff data](../workspace-diff-data.md) by reading the body as raw text. Whitespace-only text renders `(no changes)` in `#diff-tree`; fetch rejection or response-body read rejection renders `failed to load`; both paths leave `#diff-view` on the empty selected-file prompt.
  12. When the response body contains non-whitespace unified diff text, the Diff pane loader parses it with the third-party Diff2Html parser. A zero-file parse renders `(no changes)` and creates no parsed-file cache. A non-empty parse stores the parsed file array as [dashboard parsed diff file cache](../dashboard-parsed-diff-file-cache.md) on `#diff-tree._files`.
  13. The loader passes the parsed files to the [dashboard diff file tree builder](../concepts/dashboard-diff-file-tree-builder.md). For each parsed file, the builder chooses `newName` unless it is missing or `/dev/null`, otherwise uses `oldName`; coerces the chosen path to a string; splits only on `/`; creates or reuses nested directory nodes; and appends a changed-file leaf containing the displayed basename, original parsed-file index, added-line count, and deleted-line count.
  14. The Diff pane renderer replaces `#diff-tree` with generated [diff directory toggle](../gui/screens/groom-dashboard.md#diff-directory-toggle) rows and [diff file row](../gui/screens/groom-dashboard.md#diff-file-row) leaves. Directory names are rendered before file leaves, directory keys are sorted by JavaScript object-key sort order, sibling file leaves are sorted by displayed basename with `localeCompare`, and inserted directory/file names are HTML-escaped before entering the generated row HTML.
  15. The operator may activate [toggle diff directory](../gui/screens/groom-dashboard.md#toggle-diff-directory). The delegated diff-tree listener toggles only that directory node's local `collapsed` class, changing visual visibility of its child subtree without fetching data, reparsing the diff, selecting a file, changing repository state, or updating `aria-expanded`.
  16. The operator activates [select diff file row](../gui/screens/groom-dashboard.md#select-diff-file-row). The dashboard removes `active` from every currently active changed-file row in `#diff-tree`, adds `active` to the clicked row, reads its `data-file-idx`, converts it to a number, and uses that index to retrieve exactly one parsed file entry from `#diff-tree._files`.
  17. [Select diff file row](../gui/screens/groom-dashboard.md#select-diff-file-row) replaces `#diff-view` synchronously with Diff2Html output for a one-element parsed-file array. Rendering uses `drawFileList: false`, line matching, line-by-line output, and dark color scheme, so the right pane shows only the selected file's diff and does not send another HTTP request, reparse the raw unified diff text, mutate server state, change browser URL, or send a websocket message.
- end: the dashboard remains in Diff mode with the selected repository label visible in both repository picker buttons, `#diff-tree` showing the latest loaded changed-file tree, directory collapsed states held only in local DOM classes, at most one changed-file row visually marked `active`, and `#diff-view` showing either the selected file's dark line-by-line Diff2Html render, the empty selected-file prompt, `(no changes)` in the tree, or `failed to load` after a client-side fetch/body-read failure. Server-side sidecar and fallback diff failures are represented as empty [workspace diff data](../workspace-diff-data.md), not endpoint-specific error responses.
